Added internal scaffold layer for window content management.

The new `toga.Scaffold` class provides an abstraction layer between window content assignment and backend-specific layout coordination. This enables future toolbar support and improved content management across all platforms.

- Applications can now assign `Scaffold` instances directly to `window.content`
- Backward compatibility maintained: assigning widgets to `window.content` still works
- Internal `_scaffold` property provides access to the current scaffold
- All backends (Dummy, Cocoa, GTK, Qt, WinForms, iOS, Android, Textual, Web) support scaffold-based layout
